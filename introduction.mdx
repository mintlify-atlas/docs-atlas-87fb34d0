# Subsquid Pipes SDK

<Warning>
The SDK is experimental. Expect rapid iteration and occasional breaking changes.
</Warning>

Subsquid Pipes is a TypeScript-first toolkit for streaming blockchain data, transforming it in-flight, and delivering the results to your own systems. It provides a composable architecture that glues together sources, transforms, and targets to create powerful data pipelines.

## How It Works

Every pipeline is described as a composition of pieces via the `pipe()` helper. You can run the same code in CLIs, backend services, or any Node.js environment:

```typescript
const stream = evmPortalSource({ portal: 'https://portal.sqd.dev/datasets/ethereum-mainnet' })
  .pipe(evmDecoder({ events: { transfers: commonAbis.erc20.events.Transfer } }))
  .pipeTo(clickhouseTarget({ client, onData: async ({ data }) => { /* process */ } }))
```

## Sources

Sources connect to Subsquid Portal datasets to stream blockchain data:

- **EVM chains**: Ethereum, Base, Polygon, Arbitrum, and more
- **Solana**: Mainnet and devnet
- **Hyperliquid**: DEX data

All sources provide consistent streaming interfaces with automatic batching and checkpoint management.

## Transforms

Transforms decode raw blockchain data into typed objects:

- **EVM Decoder**: Decode events, transactions, and traces using ABIs
- **Solana Decoder**: Parse instructions, accounts, and program data
- **Custom Transforms**: Build your own transformation logic with the pipe API

Every decoder supports filtering by block range, contract addresses, and event signatures.

## Targets

Targets persist or forward your processed data:

- **ClickHouse**: High-performance analytical database with automatic offset management
- **PostgreSQL**: Via Drizzle ORM for type-safe schema definitions
- **Custom Targets**: Implement your own sink by extending the `createTarget` API

All targets handle fork reconciliation and provide transactional semantics.

## Observability

Built-in tools for production monitoring:

- **Profiler**: Measure performance of each pipeline stage
- **Structured Logging**: Built on Pino for high-performance logging
- **Prometheus Metrics**: Export metrics for monitoring and alerting
- **Custom Metrics**: Add your own application-specific metrics
