# Quickstart Guide

This guide will walk you through creating your first Subsquid Pipes pipeline to stream and decode ERC-20 transfer events from Ethereum Mainnet.

## Prerequisites

Before you begin, make sure you have:

- Node.js 22.15.0 or higher installed
- Basic understanding of TypeScript
- A code editor

## Step-by-Step Tutorial

<Steps>
  <Step title="Create a new project">
    Initialize a new Node.js project:

    ```bash
    mkdir my-pipeline
    cd my-pipeline
    pnpm init
    ```

    Add a `"type": "module"` field to your `package.json` to enable ES modules.
  </Step>

  <Step title="Install Subsquid Pipes">
    Install the SDK using your preferred package manager:

    <CodeGroup>
    ```bash pnpm
    pnpm add @subsquid/pipes
    ```

    ```bash npm
    npm install @subsquid/pipes
    ```

    ```bash yarn
    yarn add @subsquid/pipes
    ```
    </CodeGroup>
  </Step>

  <Step title="Create your first pipeline">
    Create a new file called `src/erc20-transfers.ts` with the following code:

    ```typescript src/erc20-transfers.ts
    import { commonAbis, evmDecoder, evmPortalSource } from '@subsquid/pipes/evm'

    async function main() {
      const stream = evmPortalSource({
        portal: 'https://portal.sqd.dev/datasets/ethereum-mainnet',
      }).pipe(
        evmDecoder({
          profiler: { id: 'erc20-transfers' },
          range: { from: '12,000,000' },
          events: {
            transfers: commonAbis.erc20.events.Transfer,
          },
        }),
      )

      for await (const { data } of stream) {
        console.log(`parsed ${data.transfers.length} transfers`)
      }
    }

    void main()
    ```

    This pipeline does three things:
    1. **Connects to Ethereum Mainnet** via Subsquid Portal
    2. **Decodes ERC-20 Transfer events** starting from block 12,000,000
    3. **Streams the results** and logs the count of transfers in each batch
  </Step>

  <Step title="Run the pipeline">
    Execute your pipeline using `tsx`:

    ```bash
    pnpm dlx tsx src/erc20-transfers.ts
    ```

    You should see output like:

    ```
    parsed 847 transfers
    parsed 1,203 transfers
    parsed 956 transfers
    ...
    ```

    <Note>
    The pipeline will continue streaming new blocks as they are added to the chain.
    </Note>
  </Step>
</Steps>

## Understanding the Code

Let's break down what each part does:

### Portal Source

```typescript
evmPortalSource({
  portal: 'https://portal.sqd.dev/datasets/ethereum-mainnet',
})
```

The `evmPortalSource` connects to Subsquid's managed data infrastructure. It streams blockchain data efficiently without requiring your own node.

### EVM Decoder

```typescript
evmDecoder({
  profiler: { id: 'erc20-transfers' },
  range: { from: '12,000,000' },
  events: {
    transfers: commonAbis.erc20.events.Transfer,
  },
})
```

The decoder:
- **Filters events** by the ERC-20 Transfer signature
- **Starts from block 12,000,000** (you can also use `'latest'` for real-time data)
- **Profiles performance** to help you optimize your pipeline
- **Decodes events** into strongly-typed TypeScript objects

### Processing the Stream

```typescript
for await (const { data } of stream) {
  console.log(`parsed ${data.transfers.length} transfers`)
}
```

The stream is an async iterator that yields batches of decoded events. Each batch contains:
- `data.transfers`: Array of decoded Transfer events
- Metadata about the batch (block range, timestamps, etc.)

## Accessing Event Data

Replace the console.log with actual data processing:

```typescript
for await (const { data } of stream) {
  for (const transfer of data.transfers) {
    console.log({
      from: transfer.event.from,
      to: transfer.event.to,
      value: transfer.event.value.toString(),
      contract: transfer.contract,
      blockNumber: transfer.blockNumber,
      timestamp: transfer.timestamp,
    })
  }
}
```

## Filtering by Contract

To index only specific tokens, add a `contracts` filter:

```typescript
evmDecoder({
  range: { from: 'latest' },
  contracts: [
    '0xa0b86991c6218b36c1d19d4a2e9eb0ce3606eb48', // USDC
    '0xdac17f958d2ee523a2206206994597c13d831ec7', // USDT
  ],
  events: {
    transfers: commonAbis.erc20.events.Transfer,
  },
})
```

## Persisting Data

To save data to a database instead of just logging it, use a target:

<CodeGroup>

```typescript ClickHouse
import { createClient } from '@clickhouse/client'
import { clickhouseTarget } from '@subsquid/pipes/targets/clickhouse'

const client = createClient({
  url: 'http://localhost:10123',
  username: 'default',
  password: 'default',
})

await evmPortalSource({
  portal: 'https://portal.sqd.dev/datasets/ethereum-mainnet',
})
  .pipe(
    evmDecoder({
      range: { from: 'latest' },
      events: { transfers: commonAbis.erc20.events.Transfer },
    }),
  )
  .pipeTo(
    clickhouseTarget({
      client,
      onData: async ({ data }) => {
        console.log(`Writing ${data.transfers.length} transfers to ClickHouse`)
        // Insert logic here
      },
    }),
  )
```

```typescript PostgreSQL
import { drizzle } from 'drizzle-orm/node-postgres'
import { drizzleTarget } from '@subsquid/pipes/targets/drizzle/node-postgres'

const db = drizzle(connectionString)

await evmPortalSource({
  portal: 'https://portal.sqd.dev/datasets/ethereum-mainnet',
})
  .pipe(
    evmDecoder({
      range: { from: 'latest' },
      events: { transfers: commonAbis.erc20.events.Transfer },
    }),
  )
  .pipeTo(
    drizzleTarget({
      db,
      onData: async ({ data, ctx }) => {
        await ctx.db.insert(transfersTable).values(
          data.transfers.map(t => ({
            from: t.event.from,
            to: t.event.to,
            value: t.event.value.toString(),
          }))
        )
      },
    }),
  )
```

</CodeGroup>
