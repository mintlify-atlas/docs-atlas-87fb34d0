---
title: Solana Query Builder
description: Build type-safe queries for Solana blockchain data
---

The `SolanaQueryBuilder` class provides a fluent API for constructing type-safe queries to request specific Solana blockchain data from the Portal network. It allows you to specify which data types to fetch (instructions, transactions, logs, etc.), which fields to include, and what filters to apply.

## Class Definition

```typescript
class SolanaQueryBuilder<F extends solana.FieldSelection = {}> extends QueryBuilder<F, solana.DataRequest>
```

## Creating a Query Builder

```typescript
import { SolanaQueryBuilder } from '@subsquid/pipes/solana'

const query = new SolanaQueryBuilder()
```

## Methods

### addFields

Specify which fields to include in the returned data. This method is type-safe and will only accept valid Solana field selections.

```typescript
addFields<T extends Subset<T, solana.FieldSelection>>(fields: T): SolanaQueryBuilder<F & T>
```

**Example:**

```typescript
const query = new SolanaQueryBuilder()
  .addFields({
    block: {
      number: true,
      hash: true,
      timestamp: true,
    },
    transaction: {
      transactionIndex: true,
      signatures: true,
    },
    instruction: {
      programId: true,
      data: true,
      accounts: true,
      instructionAddress: true,
    },
  })
```

### addInstruction

Add an instruction query to fetch program instructions matching specific criteria.

```typescript
addInstruction(options: RequestOptions<solana.InstructionRequest>): this
```

**Parameters:**

<ParamField path="options.range" type="Range" required>
  Block range to query. Example: `{ from: 340000000 }` or `{ from: 340000000, to: 350000000 }`
</ParamField>

<ParamField path="options.request" type="solana.InstructionRequest" required>
  Instruction query parameters including `programId`, discriminators, and additional options.
</ParamField>

**Instruction Request Fields:**

- `programId`: Program ID(s) to filter instructions
- `d1`, `d2`, `d4`, `d8`: Instruction discriminators (use one)
- `isCommitted`: Only include committed instructions
- `innerInstructions`: Include inner (CPI) instructions
- `transaction`: Include parent transaction data
- `transactionTokenBalances`: Include token balance changes

**Example:**

```typescript
query.addInstruction({
  range: { from: 340_000_000 },
  request: {
    programId: ['whirLbMiicVdio4qvUfM5KAg6Ct8VwpYzGff3uctyCc'],
    d8: ['0xf8c69e91e17587c8'], // Swap instruction discriminator
    isCommitted: true,
    innerInstructions: true,
    transaction: true,
    transactionTokenBalances: true,
  },
})
```

### includeAllBlocks

Include all blocks in the specified range, even if they don't match other query criteria.

```typescript
includeAllBlocks(range?: Range): this
```

**Example:**

```typescript
query.includeAllBlocks({ from: 340_000_000 })
```

### addRange

Add a block range to the query (inherited from base QueryBuilder).

```typescript
addRange(range: Range): this
```

**Example:**

```typescript
query.addRange({ from: 340_000_000, to: 350_000_000 })
```

## Complete Examples

### Basic Instruction Query

```typescript
import { solanaPortalSource, SolanaQueryBuilder } from '@subsquid/pipes/solana'

const query = new SolanaQueryBuilder()
  .addFields({
    block: {
      number: true,
      timestamp: true,
    },
    instruction: {
      programId: true,
      data: true,
      accounts: true,
    },
  })
  .addInstruction({
    range: { from: 340_000_000 },
    request: {
      programId: ['whirLbMiicVdio4qvUfM5KAg6Ct8VwpYzGff3uctyCc'],
      isCommitted: true,
      transaction: true,
    },
  })

const stream = solanaPortalSource({
  portal: 'https://portal.sqd.dev/datasets/solana-mainnet',
  query,
})

for await (const { data } of stream) {
  for (const block of data.blocks) {
    console.log(`Block ${block.header.number}:`, block.instructions.length)
  }
}
```

## Field Selection

The query builder uses TypeScript's type system to ensure only valid fields are selected. Available field categories:

### Block Fields

- `number`: Block number
- `hash`: Block hash
- `timestamp`: Block timestamp
- `parentSlot`: Parent slot number
- `parentHash`: Parent block hash

### Transaction Fields

- `transactionIndex`: Index within block
- `signatures`: Transaction signatures
- `fee`: Transaction fee
- `err`: Error information
- `computeUnitsConsumed`: Compute units used

### Instruction Fields

- `programId`: Program ID
- `data`: Instruction data (base64)
- `accounts`: Account addresses
- `instructionAddress`: Hierarchical instruction address
- `transactionIndex`: Parent transaction index

### Token Balance Fields

- `account`: Token account address
- `transactionIndex`: Parent transaction index
- `preMint`: Pre-transaction mint address
- `preAmount`: Pre-transaction amount
- `preDecimals`: Pre-transaction decimals
- `postMint`: Post-transaction mint address
- `postAmount`: Post-transaction amount
- `postDecimals`: Post-transaction decimals

### Log Fields

- `programId`: Program that emitted the log
- `kind`: Log kind (e.g., 'data', 'invoke')
- `message`: Log message
- `instructionAddress`: Associated instruction address

## Type Safety

The query builder provides full type safety. The returned data type is inferred from the fields you select:

```typescript
const query = new SolanaQueryBuilder()
  .addFields({
    block: {
      number: true,
      timestamp: true,
    },
  })

// TypeScript knows that blocks will have 'number' and 'timestamp' fields
for await (const { data } of stream) {
  for (const block of data.blocks) {
    const num: number = block.header.number
    const ts: number = block.header.timestamp
    // block.header.hash // TypeScript error - not selected
  }
}
```
